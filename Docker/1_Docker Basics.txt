Docker  ->  Image, Registry, Container
        ->  Build, Ship,     Run


Docker  ->  Image = Packaging APPs Cross OS, Cross Platform, image is having layers->files, directories, metadata, apps, no driver
                    Image is made of small Linux Kernel and other layers using dockerfile, docker push to registry, docker pull to use.
                    
            Registry = Application/pkg distribution, like dockerHub, gitHub, gitLab.
            
            Container = we can download/pull a image and run multiple isolated containers on the basis of a single image.


Commands:
            docker run httpd
            docker run -d httpd
            docker run -d -p 8081:80 httpd

            docker container run httpd
            docker container run -d -p 8081:80 httpd
            docker container run -d -p 8082:80 --name web-server1 httpd
            docker container run -d -p 8083:80 --name web-server2 httpd
            
            docker pull <image-name>
            docker pull nginx


            #Flow for Docker Container: Check local image cache -> pull if not present -> create container -> run container.
